# Placey — Infrastructure Specification

## Overview

This document details the AWS infrastructure configuration for Placey, a proximity-based place discovery application. It covers networking, compute, data, security, observability, and frontend hosting.

---

## Networking

### VPC

- CIDR: `10.0.0.0/16`
- DNS hostnames: enabled
- DNS resolution: enabled

### Subnets

| Subnet                       | Type    | CIDR          | Purpose                     |
| ---------------------------- | ------- | ------------- | --------------------------- |
| Private Subnet - App (AZ-a)  | Private | `10.0.1.0/24` | Lambda functions            |
| Private Subnet - App (AZ-b)  | Private | `10.0.2.0/24` | Lambda functions (multi-AZ) |
| Private Subnet - Data (AZ-a) | Private | `10.0.3.0/24` | RDS, RDS Proxy              |
| Private Subnet - Data (AZ-b) | Private | `10.0.4.0/24` | RDS standby (multi-AZ)      |

### VPC Endpoints

| Endpoint        | Type      | Purpose                                 |
| --------------- | --------- | --------------------------------------- |
| Secrets Manager | Interface | Lambda fetches DB credentials privately |

## Security Groups

### Lambda Security Group (`sg-lambda`)

| Direction | Protocol | Port | Source/Destination | Purpose                |
| --------- | -------- | ---- | ------------------ | ---------------------- |
| Outbound  | TCP      | 5432 | `sg-rds-proxy`     | Connect to RDS Proxy   |
| Outbound  | TCP      | 443  | VPC Endpoint       | Access Secrets Manager |

### RDS Proxy Security Group (`sg-rds-proxy`)

| Direction | Protocol | Port | Source/Destination | Purpose                        |
| --------- | -------- | ---- | ------------------ | ------------------------------ |
| Inbound   | TCP      | 5432 | `sg-lambda`        | Accept connections from Lambda |
| Outbound  | TCP      | 5432 | `sg-rds`           | Forward to RDS                 |

### RDS Security Group (`sg-rds`)

| Direction | Protocol | Port | Source/Destination | Purpose                           |
| --------- | -------- | ---- | ------------------ | --------------------------------- |
| Inbound   | TCP      | 5432 | `sg-rds-proxy`     | Accept connections from RDS Proxy |

## IAM Roles

### Lambda Execution Role (`placey-lambda-role`)

Policies:

- `AWSLambdaVPCAccessExecutionRole` — ENI management for VPC
- `AWSLambdaBasicExecutionRole` — CloudWatch Logs
- Custom inline policy:
  - `secretsmanager:GetSecretValue` on the DB credentials secret ARN
  - `rds-db:connect` on the RDS Proxy resource ARN

### RDS Proxy Role (`placey-rds-proxy-role`)

Policies:

- Custom inline policy:
  - `secretsmanager:GetSecretValue` on the DB credentials secret ARN

## Compute

### Lambda Functions

- Runtime: Node.js 24.x (LTS, supported until April 2028)
- Memory: 256 MB (default, tunable per function)
- Timeout: 30 seconds
- VPC: deployed in private app subnets
- One function per API endpoint
- Environment variables: `DB_SECRET_ARN`, `DB_PROXY_ENDPOINT` (AWS_REGION is auto-injected)

### API Gateway

- Type: HTTP API (v2) — cheaper and faster than REST API for simple use cases
- Stage: `dev`, default API Gateway URL (`https://<id>.execute-api.us-east-1.amazonaws.com/dev`), no custom domain for MVP
- CORS: enabled for frontend domain
- Throttling: 1000 requests/second burst, 500 sustained (default)
- Routes map to individual Lambda functions

---

## Data

### RDS PostgreSQL + PostGIS

- Engine: PostgreSQL 16
- Instance class: `db.t3.micro` (MVP)
- Storage: 20 GB gp3
- Multi-AZ: no (MVP)
- Extensions: PostGIS
- Database name: `placey`
- Port: 5432
- Backup retention: 7 days
- Deletion protection: disabled (MVP)

### RDS Proxy

- Engine family: PostgreSQL
- Idle client timeout: 1800 seconds
- Max connections percent: 100
- Connection borrow timeout: 120 seconds
- Auth: Secrets Manager (IAM auth optional)

## Secrets Manager

### DB Credentials Secret (`placey/db-credentials`)

```json
{
  "username": "placey_admin",
  "password": "<auto-generated>",
  "host": "<rds-endpoint>",
  "port": 5432,
  "dbname": "placey"
}
```

- Auto-rotation: disabled (MVP), recommended for production
- Accessed by: Lambda (via VPC Endpoint), RDS Proxy

## Frontend

### S3

- Bucket: `placey-frontend-<account-id>`
- Static website hosting: disabled (served via CloudFront)
- Public access: blocked
- Versioning: enabled

### CloudFront

- Origin: S3 bucket (OAC — Origin Access Control)
- Default root object: `index.html`
- Custom error response: 403/404 → `index.html` (SPA routing)
- Price class: PriceClass_100 (North America + Europe)
- HTTPS: required (default CloudFront certificate for MVP)
- Domain: default CloudFront URL (`https://<id>.cloudfront.net`), no custom domain for MVP

## Observability

### CloudWatch Logs

- Log groups: one per Lambda function (`/aws/lambda/<function-name>`)
- Retention: 14 days (MVP)

### CloudWatch Alarms

| Alarm           | Metric                  | Threshold    | Period |
| --------------- | ----------------------- | ------------ | ------ |
| API 5xx errors  | API Gateway 5XXError    | > 5 in 5 min | 5 min  |
| Lambda errors   | Lambda Errors           | > 3 in 5 min | 5 min  |
| Lambda duration | Lambda Duration         | > 10s avg    | 5 min  |
| RDS CPU         | RDS CPUUtilization      | > 80%        | 5 min  |
| RDS connections | RDS DatabaseConnections | > 80         | 5 min  |
